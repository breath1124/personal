# person — 个人开发目录（Agent 约定）

本目录用于个人开发与试验；当前优先目标是搭建个人网站。

## 目录与项目约定

- 新项目放在独立子目录中（推荐：`projects/<project-name>/`），避免把多个项目混在仓库根目录。
- 每个项目必须包含 `README.md`，至少说明：项目用途、依赖安装、运行/构建、测试、部署（如有）。
- 通用脚本放在 `scripts/`；通用文档放在 `docs/`；静态资源按项目自身结构组织。

## 个人网站（默认偏好）

- 默认先做静态站点（HTML/CSS + 少量 JS），要求使用Vue或者React框架（如 Next.js、Astro、Vue 等）。
- 关注移动端体验、可访问性（a11y）与加载性能；尽量减少依赖与构建复杂度。
- 站点文案以简体中文为主；如需要可补充英文版本（可选）。

## 工程与质量

- 变更保持“小而可回滚”，优先修复根因而不是打补丁。
- 若项目已有测试体系，新增功能应补充最小必要测试；若没有，不强行引入大型测试框架。
- 不要新增格式化/脚手架工具，除非能显著降低维护成本且我同意。

## 安全与隐私

- 不要提交任何密钥、令牌、Cookie、个人隐私数据或私有链接。
- 需要配置项时使用 `.env`（不提交），并提供 `.env.example`（可提交）作为示例。

## 协作偏好（请遵守）

- 不要创建 `git commit` 或改写历史，除非我明确要求。
- 需要联网、安装依赖、运行长时间/高资源命令时，先说明并请求授权。
- 在进行结构性重构或引入新技术栈前，先给出 2–3 个可选方案并让我确认。
